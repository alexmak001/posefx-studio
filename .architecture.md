# posefx-studio — Architecture & Implementation Status

## Pipeline Overview

```
webcam/video → [frame] → PartyEngine:
                           ├─ resize frame (inference_scale 0.5)
                           ├─ pose estimator  → PoseResult  (if renderer needs it)
                           ├─ segmenter       → MaskResult  (if renderer needs it)
                           ├─ scale results back to full res
                           └─ active renderer → composited frame
                         audio capture → bass_energy ──────────┘
                                                        ↓
                                              preview window + FPS/effect HUD
```

## Module Map

### src/io/ — Input/Output
- **webcam.py** — `WebcamCapture`: OpenCV camera wrapper. Configurable device, resolution, FPS.
- **video_input.py** — `VideoFileInput`: Opens MP4/video files, loops on end. Same interface as WebcamCapture.
- **preview.py** — `PreviewWindow`: OpenCV display window with quit detection (q/ESC). Exposes `last_key` for keyboard input.
- **recorder.py** — *(stub)* Future video recording.

### src/inference/ — Model Inference
- **base.py** — Abstract interfaces: `BasePoseEstimator`, `BaseSegmenter`. Data contracts: `PoseResult`, `MaskResult`.
- **pose_estimator.py** — `YOLOPoseEstimator(BasePoseEstimator)`: YOLOv8n-pose wrapper. Returns COCO 17-keypoint format.
- **segmenter.py** — `YOLOSegmenter(BaseSegmenter)`: YOLOv8n-seg wrapper. Filters to person class (COCO class 0).

### src/audio/ — Audio Input
- **capture.py** — `AudioCapture`: Runs in background thread. Supports mic input (PyAudio) or WAV file. Performs FFT, extracts bass band (20-200Hz), applies smoothing with noise gate. Exposes `bass_energy` float (0.0-1.0).

### src/render/ — Rendering
- **base.py** — `BaseRenderer` ABC and `RenderContext` dataclass. Each renderer declares `needs_pose` / `needs_mask`.
- **utils.py** — Shared utilities: `get_head_mask(pose, frame_shape)` creates circular mask around detected head keypoints; `composite_head(output, original, pose, shape)` blends real face back into effect frame.
- **effects/__init__.py** — Shared constants: `SKELETON_CONNECTIONS` (COCO pairs), `CONFIDENCE_MIN` (0.3).
- **effects/** — 10 effect renderers (see Effects section below).

### src/engine.py — PartyEngine
- Manages active renderer, inference models, and frame processing.
- `process_frame(frame)`: downscales for inference (configurable `inference_scale`), runs only needed models, scales results back to full res, calls active renderer.
- `set_renderer(name)`, `next_renderer()`, `prev_renderer()`: thread-safe effect switching.
- `set_bass_energy(energy)`: called from audio thread each frame.

### src/utils/ — Utilities
- **config.py** — `load_config()` → `AppConfig` dataclass tree. Sections: camera, inference, debug, effects, audio.
- **timing.py** — `FPSCounter`: Rolling average FPS over 30 frames.
- **platform.py** — `detect_platform()` → 'mac' (MPS), 'jetson' (CUDA), or 'cpu'.
- **geometry.py** — *(stub)*

### src/tracking/ — *(stubs, not yet implemented)*
### src/logic/ — *(stubs, not yet implemented)*

### src/main.py — Entry Point
- Parses CLI args: `--config`, `--input` (video file), `--audio` (WAV file).
- Instantiates input source, PartyEngine, AudioCapture, PreviewWindow.
- Main loop: read frame → feed bass energy → engine.process_frame → HUD overlays (FPS, effect name, bass meter) → preview.
- Keyboard: `n`/`p` cycle effects, `b` toggle bass meter, `q`/ESC quit.

## Data Flow Contracts

| Dataclass | Fields | Shape Info |
|-----------|--------|------------|
| `PoseResult` | keypoints, confidences, boxes, num_people | (N,17,2), (N,17), (N,4), int |
| `MaskResult` | masks, combined_mask, num_people | (N,H,W), (H,W), int |
| `RenderContext` | frame, pose, mask, bass_energy, timestamp | BGR ndarray, optional results, float, float |

## Config Structure (config/demo.yaml)

- `camera` — device_id, width, height, fps
- `inference` — pose_model, seg_model, device, confidence_threshold, inference_scale
- `effects` — default effect name
- `audio` — enabled, device_index, sample_rate, chunk_size, bass band, smoothing, sensitivity, noise_gate
- `debug` — show_skeleton, show_mask, show_fps, show_keypoint_confidence
